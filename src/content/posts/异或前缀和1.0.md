## Description
给你两个整数 $n$ 和 $x$。

考虑序列 $[1, 2, 3, \dots, n]$。你需要找出包含 $x$ 且异或和为 0 的子区间数量。换句话说，就是统计满足 $(l, r)$ 的对数，使得 $1 \le l \le x \le r \le n$ 且 $l \oplus (l + 1) \oplus \dots \oplus r = 0$，其中 $\oplus$ 表示按位异或。

举个例子，若 $n = 7$ 且 $x = 6$，那么以下区间符合要求：

*   $(4, 7)$，因为 $x$ 在区间内且 $4 \oplus 5 \oplus 6 \oplus 7 = 0$；
*   $(1, 7)$，因为 $x$ 在区间内且 $1 \oplus 2 \oplus 3 \oplus 4 \oplus 5 \oplus 6 \oplus 7 = 0$。

答案可能很大，输出结果对 $998\,244\,353$ 取模。

## 输入格式

输入包含多组测试数据。第一行是测试组数 $t$ ($1 \le t \le 2 \cdot 10^5$)。接下来是每组测试的数据。

每组测试包含一行，两个整数 $n$ 和 $x$ ($1 \le x \le n \le 10^{18}$)。

## 输出格式

对每组测试，输出一个整数 —— 满足条件的区间数量对 $998\,244\,353$ 取模的结果。

## Sample 1

**Input**

```text
5
5 5
8 1
15 8
10 10
5989566119 1996588700

```

**Output**

```text
1
2
10
0
99996
```

## 题解思路
令 $P[i] = 1 \oplus 2 \oplus \dots \oplus i$ 为前缀异或和（规定 $P[0] = 0$）。

根据异或的性质，区间 $[l, r]$ 的异或和可以表示为：

$$
l \oplus (l + 1) \oplus \dots \oplus r = Pre[r] \oplus Pre[l - 1]
$$

题目要求区间异或和为 $0$，即
$$
P[r] \oplus Pre[l - 1] = 0
$$
这等价于 
$$
Pre[r] = Pre[l - 1]
$$

* 同时，题目要求 $x$ 必须在区间 $[l, r]$ 内，即 $1 \le l \le x \le r \le n$。


* 所以问题转化为了：在 $[0, x - 1]$ 中选一个 $L$，在 $[x, n]$ 中选一个 $r$，使得 $Pre[L] = Pre[r]$，求有多少种选法。


我们把 $P[i]$ 的前几项打印：

- $P[0] = 0$
- $P[1] = 1$
- $P[2] = 3$
- $P[3] = 0$
- $P[4] = 4$
- $P[5] = 1$
- $P[6] = 7$
- $P[7] = 0$
- $P[8] = 8$
- $P[9] = 1$

所以说
对于任何数字 $k$，$P[k]$ 的值严格遵循 $k \pmod 4$ 的规律：

*   如果 $k \pmod 4 = 0$，则 $P[k] = k$
*   如果 $k \pmod 4 = 1$，则 $P[k] = 1$
*   如果 $k \pmod 4 = 2$，则 $P[k] = k + 1$
*   如果 $k \pmod 4 = 3$，则 $P[k] = 0$


既然只有 0 和 1 会重复，答案就变成了两部分之和：

1. 左边 $P[L] = 0$ 的个数 $\times$ 右边 $P[r] = 0$ 的个数。
2. 左边 $P[L] = 1$ 的个数 $\times$ 右边 $P[r] = 1$ 的个数。

我们可以写两个函数 ，分别快速计算在区间 $[0, M]$ 内，有多少个下标使得 $P[k] = 0$ 或 $P[k] = 1$。


### 那边界是什么？
对于`count_0(M)` 和 `count_1(M)`
看题目的范围，题目说子段的左端点 $l$ 的最小取值是 **1**（因为 $1 \le l \le x \le r \le n$）。

当我们想要表示区间 $[l, r]$ 的异或和时，使用的是公式：

$$
P[r] \oplus P[l - 1]
$$

如果我们要找的合法子段恰好是从**最左边的第一个数字**开始的，也就是 $l = 1$，那么代入公式就会变成：

$$
P[r] \oplus P[1 - 1] = P[r] \oplus P[0]
$$

由于我们需要这段区间的异或和为 **0**，这就要求 $P[r] = P[0]$。这就意味着，代表左边界的下标（也就是 $l - 1$）的取值范围实打实地包含了 **0**，也就是区间 $[0, x - 1]$。

### 那公式如何推导？
我们如何找到某个范围内，$k \pmod 4 == 3$ 或者 $k \pmod 4 == 1$的个数？
* 给它们加上一个常数，硬凑成 $4$ 的倍数！


### 1. 推导 $P[k] = 1$ 的个数

条件是：找 $[0, M]$ 范围内满足 $k \pmod 4 == 1$ 的 $k$ 的个数。

- **列出目标数字**： $1, 5, 9, 13, \dots$

- **平移凑倍数**： 这些数字都比 4 的倍数少 3。如果我们给每个数字都**加上 3**，它们就会变成：
$4, 8, 12, 16, \dots$ （全是 4 的倍数了！）

- **范围同步变化**： 原本我们要找的 $k$ 的范围是 $0 \le k \le M$。既然给目标数字加了 3，我们把整个考察范围也往右推 3，变成：
$3 \le k + 3 \le M + 3$

- **计算答案**： 现在问题变成了：在 $[3, M + 3]$ 这个范围内，有多少个 4 的倍数？
因为 1 和 2 都不是 4 的倍数，所以在 $[3, M + 3]$ 里找 4 的倍数，和在 $[1, M + 3]$ 里找是一模一样的。
根据我们最开始的结论，$[1, M + 3]$ 里 4 的倍数个数就是：
$$
\lfloor \frac{M + 3}{4} \rfloor
$$


## 代码部分:
```cpp
#include <bits/stdc++.h>
using namespace std;

const int MOD = 998244353;

// 从0开始到M之间有几个数是 0
long long cnt0 (long long M) {
    return (M + 1) / 4 + 1;
}
// 从1开始到M之间有几个数是 1
long long cnt1 (long long M) {
    return (M + 3) / 4;
}

void solve () {
    long long n, k;
    cin >> n >> k;

    // 计算L0 L1
    //L0 (0, k)
    //L1 (k, n)
    long long L0 = cnt0 (k - 1) % MOD;
    long long L1 = cnt1 (k - 1) % MOD;

    // R0. R1
    long long R0 = (cnt0 (n) - cnt0 (k - 1) + MOD) % MOD;
    long long R1 = (cnt1 (n) - cnt1 (k - 1) + MOD) % MOD;

    long long ans = ((L0 * R0) % MOD + (L1 * R1) % MOD) % MOD;

    cout << ans << '\n';
}

int main ()
{
    ios::sync_with_stdio (false);
    cin.tie (0);

    int t;
    cin >> t;

    while (t --) {
        solve ();
    }
    return 0;
}
```
