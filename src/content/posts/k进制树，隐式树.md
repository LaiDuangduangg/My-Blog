## 问题描述

给你两个整数 `X`、`Y`，还有一个不小于 `2` 的整数 `K`。

有一个变量 `x`，初始值为 `x = X`。你可以对 `x` 执行以下操作任意次（包括零次）：

- 选择一个整数 `y`，满足 `floor(x / K) = y` 或 `floor(y / K) = x`，然后把 `x` 的值替换成 `y`。

其中，`floor(z)` 表示不大于实数 `z` 的最大整数。

请你求出使得 `x = Y` 的最少操作次数。题目保证在给定条件下，总能用有限步数让 `x = Y`。

你需要处理 `T` 组测试数据，每组独立解答。

## 约束条件

- `1 <= T <= 2 * 10^5`
- `0 <= X, Y <= 10^18`
- `2 <= K <= 10^18`
- 所有输入均为整数。

## 输入格式

输入从标准输入读取，格式如下：

```text
T
case_1
case_2
...
case_T
```

第 `i` 组测试数据（`1 <= i <= T`）格式为：

```text
X Y K
```

## 输出格式

请按顺序输出每组测试数据的答案，每个答案占一行。

## 样例 1

**输入**

```text
4
11 9 3
0 0 2
842 180 7
1948706013487601 48019760148910476 89014537
```

**输出**

```text
2
0
7
5
```

看第一组测试数据。

通过下面的操作，`x = Y` 可以在两步内完成：

- 选择 `y = 3`。因为 `floor(x / K) = floor(11 / 3) = 3`，这个选择合法。然后把 `x` 的值改成 `3`。
- 选择 `y = 9`。因为 `floor(y / K) = floor(9 / 3) = 3`，这个选择合法。然后把 `x` 的值改成 `9`。

`x = Y` 不可能用少于两步完成，所以第一行输出 `2`。

## 题意概述

给定两个非负整数 `X, Y`，以及一个不小于 `2` 的整数 `K`。变量 `x` 初始为 `X`，一次操作可以选择一个整数 `z`，满足：

下文中的 floor(a / b) 表示向下取整。由于所有数都是非负整数，它与 C++ 中的整数除法 a / b 等价。

```text
floor(x / K) = z
```

或者：

```text
floor(z / K) = x
```

然后把 `x` 替换为 `z`。求使 `x = Y` 的最少操作次数。

数据范围：

```text
0 <= X, Y <= 10^18
2 <= K <= 10^18
```

并且测试组数最多为 `2 * 10^5`。

---

## 核心思路：

把每个非负整数看成隐式树上的节点，`floor(x / K)` 是它唯一的父亲；题目要求的最少操作次数，就是节点 `X` 与节点 `Y` 在这棵树上的距离。

---

## 1. 两种操作：

第一种操作很直接：

```text
x -> floor(x / K)
```

它会让当前数走向一个更小的数。

第二种操作满足：

```text
floor(z / K) = x
```

推导这个 z 的范围：

```text
x <= z / K < x + 1
```

两边同时乘以 K：

```text
x * K <= z < (x + 1) * K
```

因为 z 是整数，所以：

```text
z in [x * K, x * K + K - 1]
```

注意，上界是：

```text
(x + 1) * K - 1 = x * K + K - 1
```

而不是：

```text
x * (K + 1) - 1
```

也就是说，从 x 出发，第二种操作一共有 K 个候选结果：

```text
x * K, x * K + 1, ..., x * K + K - 1
```

---

## 2. 模拟思路：

注意到每次除以 K 都会让数快速缩小。即使 K = 2，10^18 也只需要大约 60 次整除就能到达 0。

因此，一个很自然的想法是：

1. 如果 X > Y，先让 X 不断除以 K，直到 X <= Y；
2. 然后再从较小的数向上选择某个孩子，尝试到达 Y。

问题出在第二步：每次向孩子走都有 K 个选择，而 K 最大可以达到 10^18，根本无法枚举。

更重要的是，数值从“大于 Y”变成“小于 Y”，并不表示已经到达正确的转折点。

例如：

```text
X = 10，Y = 9，K = 2
```

如果先除一次：

```text
10 -> 5
```

此时虽然 `5 < 9`，但 `5` 并不是 `9` 的祖先，不能从这里直接向下到达 `9`。

两者的父亲链分别是：

```text
10 -> 5 -> 2 -> 1 -> 0
 9 -> 4 -> 2 -> 1 -> 0
```

真正的最短路径是：

```text
10 -> 5 -> 2 -> 4 -> 9
```

所以这道题不是在数轴上逼近目标。我们需要找到两个数真正的公共转折点。

---

## 3. 把操作关系看成一棵树

对于任意正整数 v，定义：

```text
parent(v) = floor(v / K)
```

那么：

- 每个正整数 v 只有一个父亲；
- v 的孩子是：

  `v * K, v * K + 1, ..., v * K + K - 1`

- 当 v > 0 时，parent(v) < v；
- 不断寻找父亲，最终一定会到达 0。

因此，所有非负整数形成了一棵以 0 为根的隐式树。严格来说，parent(0) = 0 会产生一个没有意义的自环，在最短路中直接忽略即可。

题目的一次操作，就是在这棵树的一条父子边上走一步：

- 第一种操作：从孩子走向父亲；
- 第二种操作：从父亲走向某个孩子。

所以原问题可以重新描述成：

> 求隐式树上节点 X 与节点 Y 的最短距离。

---

## 4. 为什么Y 也可以不断除以 K？

题目中真正变化的变量 x 的确从 X 开始。

但是，算法中执行：

```cpp
Y /= K;
```

并不是在模拟“真实变量 x 从 X 突然变成了 Y / K”，而是在**从终点 Y 反向追踪路径**。

令：

```text
p = floor(Y / K)
```

那么一定存在唯一的余数 r，使得：

```text
Y = p * K + r
0 <= r < K
```

因此，Y 是节点 p 的一个孩子。

题目允许在这条父子边上双向移动：

- 从孩子 Y 走向父亲 p，因为 floor(Y / K) = p；
- 从父亲 p 走回孩子 Y，仍然因为 floor(Y / K) = p。

所以，算法从目标端记录的父亲链：

```text
Y -> floor(Y/K) -> floor(floor(Y/K)/K) -> ...
```

倒过来以后，就是一条完全合法的实际路径：

```text
公共祖先 -> ... -> Y
```

分析这么多，实际上也就是说，从`Y / k`一直到最近公共祖先这条路，也一定可以从最近公共祖先以相同的步数走回Y,因此我们可以直接用Y 向下找替代掉X向上找。

### 除法取整丢失的信息怎么办？

从 Y 变成 floor(Y / K) 的确会丢掉余数。

例如：

```text
9、10、11
```

除以 `3` 取整以后都会得到 `3`。所以如果只知道父节点 `3`，确实不能唯一恢复原来的孩子。

但是这里的终点 Y 本来就是已知的，算法也知道完整的 Y 侧祖先链。被删掉的分支信息可以写成：

```text
r = Y % K
```

倒放路径时，应该选择的孩子已经被原来的 Y 唯一确定：

```text
Y = p * K + r
```

所以准确的说法不是“整除后直接乘回来就能恢复”，而是：

> 整除会丢掉余数，但是这个 + K的弹性区间是一定可以让我们在相同的步数之内找到Y,只是我们没办法用一条唯一确定的路去找,题目也仅仅要求是计算步数。

---

## 5. 最近链路选择：

考虑样例：

```text
X = 11，Y = 9，K = 3
```

分别从两端向父亲移动：

```text
11 -> 3 -> 1 -> 0
 9 -> 3 -> 1 -> 0
```

两条祖先链第一次相交在 `3`，所以真正的最短路径是：

```text
11 -> 3 -> 9
```

算法中：

```text
9 -> 3
```

是从终点进行的反向分析。把它倒过来以后：

```text
3 -> 9
```

是合法操作，因为：

```text
floor(9 / 3) = 3
```

因此，从 `3` 向下时根本不需要在：

```text
9、10、11
```

之间猜测。目标端反向得到的祖先链，已经明确告诉我们正确分支是 `9`。

两条祖先链的第一个公共节点，在树上称为最近公共祖先。不过本题树高最多约为 60，不需要倍增等高级 LCA 算法。

---

## 6. K 进制视角

这个树模型还可以用 K 进制解释：

- 除以 K：删除 K 进制表示的最后一位；
- 变成 `K * x + r`：在末尾追加一位 r，其中 0 <= r < K。

仍以：

```text
X = 11，Y = 9，K = 3
```

为例：

```text
11 的三进制：102
 9 的三进制：100
```

操作过程相当于：

```text
102 -> 10 -> 100
```

即：

1. 从 `102` 中删除最后一位 `2`；
2. 在 `10` 后面追加目标剩余的数字 `0`。

所以最优策略可以理解为：

> 保留 X 与 Y 的最长公共前缀，删除 X 剩余的后缀，再追加 Y 剩余的后缀。

树上的最近公共祖先，就是两者最长的公共前缀。对于根节点 0，可以把它理解为空前缀。

---

## 7. 最终的实现思路

理论上，可以分别保存 X 与 Y 的祖先链，再寻找第一个公共节点。

本题还有一个更简单的实现：当两个数不相等时，每次让数值较大的那个数除以 K。

为什么这样一定正确？

假设当前：

```text
x > y
```

那么 x 不可能是 y 的祖先。因为在这棵树中，一个节点的所有严格后代都比它大；如果 x 是 y 的祖先，就应该有 y > x，与当前条件矛盾。

既然 x 不可能是 y 的祖先，那么树上从 x 到 y 的唯一简单路径，第一步必然是从 x 走向父亲：

```text
x -> floor(x / K)
```

当 y > x 时同理，让 y 走向父亲。

因此，每一步都不是凭数值大小进行猜测，而是由树上的唯一最短路径强制决定的。

算法如下：

```text
answer = 0

while X != Y:
    如果 X > Y：X = floor(X / K)
    否则：Y = floor(Y / K)
    answer++
```

当两者第一次相等时，这个节点就是它们的最近公共祖先；累计移动的边数，就是原始 X 与 Y 的树上距离。

---

## 8. 正确性说明

### 引理一：所有非负整数构成一棵以 0 为根的隐式树

每个正整数 v 都有唯一父亲 floor(v / K)，而且父亲严格小于 v，所以不断寻找父亲最终一定到达 0。因此所有正整数都唯一连接到根节点 0。

### 引理二：每次让较大的数走向父亲，都走在唯一最短路径上

若 x > y，则 x 不可能是 y 的祖先。因此从 x 到 y 的唯一简单路径必然先经过 parent(x)。所以令 x = floor(x / K) 不会偏离最短路径。y > x 时同理。

### 结论

算法的每一步都沿着两点之间唯一的简单路径移动。当 x = y 时，两侧在最近公共祖先处汇合，累计操作次数正好等于原始节点 X, Y 之间的树上距离，因此答案最小。

---

## 9. 复杂度分析

每次循环至少会让一个正数除以 K。因此单组时间复杂度可以写为：

```text
O(log_K(max(1, X)) + log_K(max(1, Y)) + 1)
```

最坏情况下 K = 2，每个数最多除大约 60 次，所以单组循环次数最多约为 120。

空间复杂度为：

```text
O(1)
```

## 代码如下:
```cpp
#include <bits/stdc++.h>
using namespace std;

void solve () {
    long long x, y, k;
    cin >> x >> y >> k;

    int cnt = 0;
    while (x != y) {
        if (x > y) {
            x /= k;
            cnt ++;
        }
        else if (y > x) {
            y /= k;
            cnt ++;
        }
    }
    cout << cnt << '\n';
    return;
}

int main ()
{
    ios::sync_with_stdio (false);
    cin.tie (0);

    int t;
    cin >> t;

    while (t --) {
        solve ();
    }
    return 0;
}
```
