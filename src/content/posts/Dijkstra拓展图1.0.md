## 题目描述

1944 年，特种兵麦克接到国防部的命令，要求立即赶赴太平洋上的一个孤岛，营救被敌军俘虏的大兵瑞恩。

瑞恩被关押在一个迷宫里。迷宫地形复杂，但幸运的是，麦克得到了一张迷宫的地形图。

迷宫的外形是一个长方形，其南北方向被划分为 $N$ 行，东西方向被划分为 $M$ 列，于是整个迷宫被划分为 $N \times M$ 个单元。

每一个单元的位置可用一个有序数对表示，即：

```txt
单元的行号，单元的列号
```

南北或东西方向相邻的 $2$ 个单元之间可能互通，也可能有一扇锁着的门，或者是一堵不可逾越的墙。

迷宫中有一些单元存放着钥匙，并且所有的门被分成 $P$ 类。打开同一类门的钥匙相同，不同类门的钥匙不同。

大兵瑞恩被关押在迷宫的东南角，即 $(N, M)$ 单元里，并且已经昏迷。迷宫只有一个入口，在西北角。也就是说，麦克可以直接进入 $(1, 1)$ 单元。

另外，麦克从一个单元移动到另一个相邻单元的时间为 $1$，拿取所在单元的钥匙的时间以及用钥匙开门的时间可以忽略不计。

试设计一个算法，帮助麦克以最快的方式到达瑞恩所在单元，营救大兵瑞恩。

## 输入格式

第 $1$ 行有 $3$ 个整数，分别表示 $N, M, P$ 的值。

第 $2$ 行是 $1$ 个整数 $K$，表示迷宫中门和墙的总数。

第 $I + 2$ 行，满足 $1 \le I \le K$，有 $5$ 个整数，依次为 $X_{i1}, Y_{i1}, X_{i2}, Y_{i2}, G_i$：

- 当 $G_i \ge 1$ 时，表示 $(X_{i1}, Y_{i1})$ 单元与 $(X_{i2}, Y_{i2})$ 单元之间有一扇第 $G_i$ 类的门。
- 当 $G_i = 0$ 时，表示 $(X_{i1}, Y_{i1})$ 单元与 $(X_{i2}, Y_{i2})$ 单元之间有一堵不可逾越的墙。

其中：

$$
|X_{i1} - X_{i2}| + |Y_{i1} - Y_{i2}| = 1
$$

并且：

$$
0 \le G_i \le P
$$

第 $K + 3$ 行是一个整数 $S$，表示迷宫中存放的钥匙总数。

第 $K + 3 + J$ 行，满足 $1 \le J \le S$，有 $3$ 个整数，依次为 $X_{i1}, Y_{i1}, Q_i$，表示第 $J$ 把钥匙存放在 $(X_{i1}, Y_{i1})$ 单元里，并且第 $J$ 把钥匙是用来开启第 $Q_i$ 类门的。

其中：

$$
1 \le Q_i \le P
$$

输入数据中，同一行各相邻整数之间用一个空格分隔。

## 输出格式

输出麦克营救大兵瑞恩所需的最短时间。

如果问题无解，则输出：

```txt
-1
```

## 样例 1

### 输入

```txt
4 4 9
9
1 2 1 3 2
1 2 2 2 0
2 1 2 2 0
2 1 3 1 0
2 3 3 3 0
2 4 3 4 1
3 2 3 3 0
3 3 4 3 0
4 3 4 4 0
2
2 1 2
4 2 1
```

### 输出

```txt
14
```

## 数据范围

$$
|X_{i1} - X_{i2}| + |Y_{i1} - Y_{i2}| = 1
$$

$$
0 \le G_i \le P
$$

$$
1 \le Q_i \le P
$$

$$
N, M, P \le 10
$$

$$
K < 150
$$

$$
S \le 14
$$

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 20;
const int MAXM = 20;
const int INF = 2e9;
vector <vector <vector <int>>> grid (MAXN, vector <vector <int>> (MAXM, vector <int> (4, -1)));

int dx[] = {1, 0, -1, 0};
int dy[] = {0, 1, 0, -1};

int get_dir (int x1, int y1, int x2, int y2) {
    int gapx = x2 - x1;
    int gapy = y2 - y1;
    for (int i = 0; i < 4; i ++) {
        if (gapx == dx[i] && gapy == dy[i]) {
            return i;
        }
    }
    return -1;
}

struct Infoma1 {
    int x;
    int y;
    int cur_keys;
};

int main ()
{
    int n, m, p, k;
    cin >> n >> m >> p >> k;

     
    for (int i = 0; i < k; i ++) {
        int x1, y1, x2, y2, g;
        cin >> x1 >> y1 >> x2 >> y2 >> g;

        int dir_1 = get_dir (x1, y1, x2, y2);
        int dir_2 = get_dir (x2, y2, x1, y1);
        grid[x1][y1][dir_1] = g;
        grid[x2][y2][dir_2] = g;
    }

    int s; 
    cin >> s;
    vector <vector <int>> keys (MAXN, vector <int> (MAXM, 0));

    for (int i = 0; i < s; i ++) {
        int x, y, kinds;
        cin >> x >> y >> kinds;

        keys[x][y] |= (1 << (kinds - 1));
    }
    
    int max_keys = 1 << p; // 钥匙从0开始，只能占到p - 1位
    vector <vector <vector<int>>> dist (MAXN, vector <vector <int>> (MAXM, vector <int> (max_keys, INF)));
    
    int start_key = keys[1][1];
    dist[1][1][start_key] = 0;
    queue <Infoma1> que;
    que.push ({1, 1, start_key}); //x, y, key

    while (!que.empty ()) {
        auto [x, y, cur_keys] = que.front ();
        que.pop ();

        for (int i = 0; i < 4; i ++) {
            int nx = x + dx[i];
            int ny = y + dy[i];

            int need_key = grid[x][y][i];
            if (nx < 1 || nx > n || ny < 1 || ny > m)  continue; // 越界
            if (grid[x][y][i] == 0)  continue; // 墙
            if (need_key > 0 && !((1 << (need_key - 1)) & cur_keys))  continue; // 没有这种房门钥匙

            int nex_k = cur_keys | keys[nx][ny];
            if (dist[nx][ny][nex_k] > dist[x][y][cur_keys] + 1) {
                dist[nx][ny][nex_k] = dist[x][y][cur_keys] + 1;
                que.push ({nx, ny, nex_k});
            }
        }
    }
    int ans = INF;
    for (int keys = 0; keys < max_keys; keys ++) {
        ans = min (ans, dist[n][m][keys]);
    }
    if (ans == INF) {
        cout << -1 << '\n';
    }
    else {
        cout << ans << '\n';
    }
    return 0;
}
```

# 题解：

## 1. 题意分析

这道题本质上是一道 **BFS 最短路问题**。

普通的迷宫 BFS 中，每个点的状态通常只需要记录当前位置：

```cpp
(x, y)
```

但是这道题不一样。

因为迷宫中有门和钥匙，能不能从某个格子走到另一个格子，不仅取决于当前位置，还取决于当前手里有哪些钥匙。

也就是说，同样是走到点 `(x, y)`，如果手里拥有的钥匙不同，那么后面能走的路也可能不同。

例如：

```text
第一次到达 (x, y)：没有 2 号钥匙，不能通过 2 号门
第二次到达 (x, y)：已经有 2 号钥匙，可以通过 2 号门
```

所以这两个状态不能看成同一个状态。

因此，本题中的 BFS 状态应该表示为：

```cpp
(x, y, 当前拥有的钥匙)
```

---

## 2. 为什么需要状态压缩？

假设一共有 5 种钥匙，分别是：

```text
1 号钥匙、2 号钥匙、3 号钥匙、4 号钥匙、5 号钥匙
```

如果直接用多维数组来记录每种钥匙是否拥有，那么可能会写成：

```cpp
dist[x][y][0][1][1][0][1]
```

其中：

```text
0 表示没有这把钥匙
1 表示拥有这把钥匙
```

例如：

```cpp
dist[x][y][0][1][1][0][1]
```

表示当前在点 `(x, y)`，并且：

```text
没有 1 号钥匙
有 2 号钥匙
有 3 号钥匙
没有 4 号钥匙
有 5 号钥匙
```

这种写法虽然比较直观，但是数组维度太多，写起来非常麻烦，也会浪费空间。

所以我们可以把“拥有钥匙的情况”压缩成一个二进制数。

这就是 **状态压缩**。

---

## 3. 如何用二进制表示钥匙状态？

因为钥匙的种类数比较小，所以可以用一个整数的二进制位来表示当前拥有了哪些钥匙。

假设有 5 种钥匙，可以这样规定：

```text
第 0 位：表示是否拥有 1 号钥匙
第 1 位：表示是否拥有 2 号钥匙
第 2 位：表示是否拥有 3 号钥匙
第 3 位：表示是否拥有 4 号钥匙
第 4 位：表示是否拥有 5 号钥匙
```

例如：

```text
10110
```

从右往左看：

```text
第 0 位是 0：没有 1 号钥匙
第 1 位是 1：有 2 号钥匙
第 2 位是 1：有 3 号钥匙
第 3 位是 0：没有 4 号钥匙
第 4 位是 1：有 5 号钥匙
```

所以 `10110` 表示当前拥有：

```text
2 号钥匙、3 号钥匙、5 号钥匙
```

没有：

```text
1 号钥匙、4 号钥匙
```

于是原来复杂的多维状态：

```cpp
dist[x][y][0][1][1][0][1]
```

就可以压缩成：

```cpp
dist[x][y][state]
```

其中：

```text
x, y 表示当前位置
state 表示当前拥有的钥匙集合
```

如果钥匙种类数是 `P`，那么所有钥匙状态一共有：

```cpp
1 << P
```

种。

也就是：

```text
2^P
```

种。

本题中 `P <= 10`，所以最多只有：

```text
2^10 = 1024
```

种钥匙状态，可以接受。

---

## 4. 如何建图？

题目中描述的是：

```text
从某个点 (x1, y1) 到另一个相邻点 (x2, y2) 之间有一扇门或一堵墙
```

所以我们可以用：

```cpp
grid[x][y][dir]
```

表示从 `(x, y)` 往 `dir` 方向走时，会遇到什么。

其中：

```text
grid[x][y][dir] = -1  表示普通道路，可以直接走
grid[x][y][dir] = 0   表示墙，不能走
grid[x][y][dir] = k   表示第 k 类门，需要第 k 类钥匙
```

这里的 `dir` 表示方向。

例如可以规定四个方向为：

```cpp
int dx[] = {1, 0, -1, 0};
int dy[] = {0, 1, 0, -1};
```

分别表示：

```text
0：向下
1：向右
2：向上
3：向左
```

---

## 5. 门和墙为什么要双向记录？

因为门和墙是在两个相邻格子之间的限制。

假设 `(x1, y1)` 和 `(x2, y2)` 之间有一扇 2 号门，并且 `(x2, y2)` 在 `(x1, y1)` 的右边：

```text
(x1, y1)  <-->  (x2, y2)
          2号门
```

那么从 `(x1, y1)` 往右走，会遇到 2 号门。

同时，从 `(x2, y2)` 往左走，也会遇到同一扇 2 号门。

所以需要双向记录：

```cpp
grid[x1][y1][右] = 2;
grid[x2][y2][左] = 2;
```

在代码中，可以先求出两个方向：

```cpp
int dir1 = get_dir(x1, y1, x2, y2);
int dir2 = get_dir(x2, y2, x1, y1);
```

然后记录：

```cpp
grid[x1][y1][dir1] = g;
grid[x2][y2][dir2] = g;
```

如果 `g = 0`，表示这两个格子之间是一堵墙。

如果 `g > 0`，表示这两个格子之间是一扇第 `g` 类门。

---

## 6. 钥匙怎么存？

每个格子上可能放着钥匙。

同样地，我们也可以用一个二进制数来表示某个格子上有哪些钥匙。

设：

```cpp
key[x][y]
```

表示格子 `(x, y)` 上存放的所有钥匙。

一开始，每个格子都没有钥匙，所以初始状态为：

```text
0000
```

如果某个格子上有一把 2 号钥匙，那么就让第 `2 - 1 = 1` 位变成 `1`：

```cpp
key[x][y] |= (1 << (2 - 1));
```

也就是：

```text
0000
|
0010
=
0010
```

如果这个格子上还有 4 号钥匙，那么继续加入：

```cpp
key[x][y] |= (1 << (4 - 1));
```

状态就会变成：

```text
0010
|
1000
=
1010
```

所以一个格子上即使有多把钥匙，也可以全部压缩到一个整数里。

输入钥匙时，可以这样写：

```cpp
keys[x][y] |= (1 << (kind - 1));
```

其中 `kind` 表示当前这把钥匙的编号。

---

## 7. BFS 状态设计

BFS 队列中存放的状态为：

```cpp
(x, y, state)
```

含义是：

```text
当前在格子 (x, y)
当前拥有的钥匙集合为 state
```

距离数组为：

```cpp
dist[x][y][state]
```

含义是：

```text
到达格子 (x, y)，并且当前钥匙状态为 state 时，所需要的最短时间
```

初始化时，所有距离都设为无穷大：

```cpp
dist[x][y][state] = INF;
```

---

## 8. BFS 初始化

起点是 `(1, 1)`。

需要注意：

如果起点本身就有钥匙，那么一开始就应该拿到这些钥匙。

所以初始钥匙状态为：

```cpp
int start_state = keys[1][1];
```

然后初始化 BFS：

```cpp
dist[1][1][start_state] = 0;
q.push({1, 1, start_state});
```

---

## 9. BFS 状态转移

每次从队列中取出一个状态：

```cpp
(x, y, state)
```

然后枚举四个方向，尝试走到新格子：

```cpp
int nx = x + dx[i];
int ny = y + dy[i];
```

转移时需要依次判断下面几种情况。

---

### 9.1 判断是否越界

如果新位置 `(nx, ny)` 不在迷宫范围内，就不能走：

```cpp
if (nx < 1 || nx > n || ny < 1 || ny > m) {
    continue;
}
```

---

### 9.2 判断是否遇到墙

如果当前方向是一堵墙，不能通过：

```cpp
if (grid[x][y][i] == 0) {
    continue;
}
```

---

### 9.3 判断是否遇到门但没有钥匙

如果：

```cpp
grid[x][y][i] > 0
```

说明这条边是一扇门。

设这扇门需要的钥匙编号为：

```cpp
int need = grid[x][y][i];
```

那么需要判断当前状态 `state` 中有没有这把钥匙。

第 `need` 类钥匙对应的二进制位是：

```cpp
1 << (need - 1)
```

判断是否拥有这把钥匙：

```cpp
state & (1 << (need - 1))
```

如果结果为 `0`，说明没有这把钥匙，不能通过这扇门：

```cpp
if (need > 0 && !(state & (1 << (need - 1)))) {
    continue;
}
```

---

### 9.4 可以走时更新钥匙状态

如果可以走到新格子 `(nx, ny)`，那么到达之后要立刻拿走这个格子上的钥匙。

新的钥匙状态为：

```cpp
int new_state = state | keys[nx][ny];
```

这里使用按位或 `|`，表示：

```text
原来拥有的钥匙 + 新格子上的钥匙
```

例如，当前已经有 2 号钥匙：

```text
state = 0010
```

新格子上有 4 号钥匙：

```text
keys[nx][ny] = 1000
```

合并之后：

```text
new_state = 0010 | 1000 = 1010
```

---

### 9.5 更新最短距离

如果这个新状态还没有被更短的路径到达过，就更新距离并入队：

```cpp
if (dist[nx][ny][new_state] > dist[x][y][state] + 1) {
    dist[nx][ny][new_state] = dist[x][y][state] + 1;
    q.push({nx, ny, new_state});
}
```

因为每次移动的时间都是 `1`，所以使用普通 BFS 就可以保证最短路。

---

## 10. 答案统计

终点是：

```cpp
(n, m)
```

题目只要求到达终点，并不要求最后必须拥有某些钥匙。

所以，到达 `(n, m)` 时，任何钥匙状态都可以作为答案。

因此需要枚举所有钥匙状态，取最小值：

```cpp
int ans = INF;

for (int state = 0; state < (1 << p); state++) {
    ans = min(ans, dist[n][m][state]);
}
```

如果所有状态都无法到达终点，说明无解：

```cpp
if (ans == INF) {
    cout << -1 << '\n';
} else {
    cout << ans << '\n';
}
```

---

## 11. 正确性说明

### 11.1 状态设计是完整的

对于之后能走哪些路，只需要知道两件事：

```text
1. 当前在哪个格子
2. 当前拥有哪些钥匙
```

当前位置决定了可以尝试走向哪些相邻格子。

钥匙状态决定了能不能通过某一类门。

所以 `(x, y, state)` 已经包含了影响后续决策的全部信息，状态设计是完整的。

---

### 11.2 状态转移是合法的

从当前状态向四周扩展时，程序会依次判断：

```text
1. 是否越界
2. 是否遇到墙
3. 是否遇到门但没有对应钥匙
4. 能走过去后，是否需要更新钥匙状态
```

只有满足所有条件的移动才会加入队列。

所以 BFS 中产生的所有状态都是合法状态。

---

### 11.3 BFS 可以保证最短路

每次从一个格子移动到相邻格子的代价都是 `1`。

因此，在状态图中，每条边的权值都是 `1`。

对于边权全部为 `1` 的图，BFS 可以保证先到达的状态距离更短。

所以 `dist[x][y][state]` 记录的就是到达该状态的最短时间。

---

### 11.4 最终答案是正确的

题目要求的是到达 `(n, m)` 的最短时间，不要求最终钥匙状态是什么。

所以只要位置是 `(n, m)`，不管当前拥有了哪些钥匙，都可以作为一种答案。

枚举所有：

```cpp
dist[n][m][state]
```

取最小值，就能得到最终答案。

---

## 12. 复杂度分析

设迷宫大小为 `N × M`，钥匙种类数为 `P`。

每个格子有 `2^P` 种钥匙状态。

所以状态总数为：

```text
N × M × 2^P
```

每个状态最多枚举 4 个方向。

因此时间复杂度为：

```text
O(N × M × 2^P × 4)
```

省略常数后为：

```text
O(N × M × 2^P)
```

距离数组需要记录每个状态的最短距离，所以空间复杂度为：

```text
O(N × M × 2^P)
```

本题数据范围较小，`P <= 10`，所以状态数量完全可以接受。

## 13. 易错点总结

### 13.1 不能只用二维数组记录访问情况

错误写法：

```cpp
visited[x][y]
```

这样会把“同一个格子但钥匙不同”的情况误认为同一个状态。

正确写法应该是：

```cpp
dist[x][y][state]
```

或者：

```cpp
visited[x][y][state]
```

---

### 13.2 起点上的钥匙也要拿

如果 `(1, 1)` 上有钥匙，一开始就应该拥有这些钥匙：

```cpp
int start_state = keys[1][1];
```

---

### 13.3 门和墙要双向建图

题目给的是两个相邻格子之间的门或墙。

所以需要同时记录两个方向：

```cpp
grid[x1][y1][dir1] = g;
grid[x2][y2][dir2] = g;
```

---

### 13.4 普通道路默认可以走

没有在输入中出现的边，默认是普通道路。

所以可以把 `grid` 初始化为 `-1`：

```cpp
grid[x][y][dir] = -1;
```

其中：

```text
-1 表示普通道路
0 表示墙
正数表示门
```

---

### 13.5 到达终点时不要求钥匙状态

终点答案不是某一个固定状态，而是所有状态中的最小值：

```cpp
min(dist[n][m][state])
```

---

## 14. 思路总结

这道题的核心思路可以概括为：

```text
1. 普通 BFS 只记录位置，但本题还要记录钥匙状态
2. 用二进制状态压缩表示当前拥有的钥匙
3. 用 grid[x][y][dir] 记录某个方向上是普通路、墙还是门
4. 用 keys[x][y] 记录每个格子上有哪些钥匙
5. BFS 时判断能不能走过墙或门
6. 走到新格子后更新钥匙状态
7. 最后枚举终点的所有钥匙状态，取最短时间
```

最终状态为：

```cpp
(x, y, state)
```

其中：

```text
(x, y) 表示当前位置
state 表示当前拥有的钥匙集合
```

这就是本题使用 **BFS + 状态压缩** 的原因。
