### 题目描述

给定一个长度为 $N$ 的正整数序列 $A = (A_1, \dots, A_N)$ 和一个长度为 $M$ 的正整数序列 $B = (B_1, \dots, B_M)$。
求以下双重求和式对 $998244353$ 取模后的结果：
$$\sum_{i=1}^N \sum_{j=1}^M A_i \cdot B_j \cdot (i \bmod j)$$

### 约束条件
* $1 \le N, M \le 5 \times 10^5$
* $1 \le A_i, B_j \le 5 \times 10^5$
* 所有输入值均为整数。

---
### 输入格式

输入从标准输入读取，格式如下：

```text
N M
A_1 A_2 ... A_N
B_1 B_2 ... B_M
```

### 输出格式

输出答案，单独一行。


### 样例 1

**Input**

```text
6 4
1 6 9 2 3 1
1 10 3 7
```

**Output**

```text
508
```


### 题解思路

这道题的数据量达到了 $5 \times 10^5$，如果使用两层 `for` 循环进行 $O(N \times M)$ 的暴力计算，运算次数高达 $2.5 \times 10^{11}$，必定会超时。因此，我们需要将含有取模运算的求和式进行**代数变形**，并利用**前缀和**与**调和级数**的思想将其优化到 $O(N \log M)$。

#### 1. 消灭取模符号
取模运算的本质是求余数。在数学上，我们可以将其转化为带有向下取整的减法：
$$i \bmod j = i - j \cdot \lfloor \frac{i}{j} \rfloor$$
代入原式，得到：
$$\sum_{i=1}^N \sum_{j=1}^M A_i \cdot B_j \cdot \left( i - j \cdot \lfloor \frac{i}{j} \rfloor \right)$$

#### 2. 分离变量（提取公因式）
将括号乘开，我们可以把原式拆分成两部分相减：

**左半部分：**
$$\sum_{i=1}^N \sum_{j=1}^M A_i \cdot B_j \cdot i = \left( \sum_{i=1}^N A_i \cdot i \right) \times \left( \sum_{j=1}^M B_j \right)$$
这里利用了乘法分配律（变量分离），把与 $i$ 相关的项和与 $j$ 相关的项完全分离开来。这部分只需要 $O(N+M)$ 的时间就能用两个独立循环算完。

**右半部分：**
$$\sum_{i=1}^N \sum_{j=1}^M A_i \cdot B_j \cdot j \cdot \lfloor \frac{i}{j} \rfloor = \sum_{j=1}^M \left( B_j \cdot j \times \sum_{i=1}^N A_i \cdot \lfloor \frac{i}{j} \rfloor \right)$$
现在的核心难点变成了如何快速求出内层的求和：$\sum_{i=1}^N A_i \cdot \lfloor \frac{i}{j} \rfloor$。

也就是说，在右边部分，我需要有两个for循环，在每一个j的情况下，算出此时的

$$\sum_{i=1}^{N} A_i \cdot \left\lfloor \frac{i}{j} \right\rfloor$$


#### 3. 调和级数与前缀和分块
对于一个固定的 $j$，$\lfloor \frac{i}{j} \rfloor$ 的值是呈**阶梯状分布**的。
具体来说，在区间 $[k \cdot j, \ (k+1) \cdot j - 1]$ 之间，$\lfloor \frac{i}{j} \rfloor$ 的值恒定为 $k$！
    
既然乘数 $k$ 在一个区间内是固定的，我们就可以提前算出 $A$ 数组的**前缀和 (Prefix Sum)**。对于每一个 $j$，我们不需要一步步遍历 $i$，而是以 $j$ 为步长进行跳跃求和：
$$\sum_{k=1}^{\lfloor N/j \rfloor} k \times \Big( Pref[\min(N, (k+1)j - 1)] - Pref[k \cdot j - 1] \Big)$$

假设 $j=4$，那么 $\displaystyle \sum_{i=1}^{N} A_i \cdot \left\lfloor \frac{i}{j} \right\rfloor$ 应该等于：


$$
\begin{aligned}
& (A_0, A_1, A_2, A_3) \times 0 \\
& (A_4, A_5, A_6, A_7) \times 1 \\
& (A_8, A_9, A_{10}, A_{11}) \times 2 \\
& \dots \dots \dots \\
& (A_{16}, A_{17}, A_{18}) \times 3 \\
\end{aligned}
$$
$$
\downarrow
$$
$$
\text{ 假设 } N \text{ 恰好到 } 18
$$
    

那么图中这一块，我们可以选择枚举 $\left\lfloor \frac{i}{j} \right\rfloor$ 的值，即 $val$ 从 $0$ 开始，那到哪里结束呢？

我们注意到每一块的长度 $= j$，

所以每块的左端点 $L = val \times j$

右端点 $R = (val + 1) \times j - 1$

因此枚举 $val$，条件令 $L = val \times j \leqslant N$ 即可


### 4.算法复杂度估计
外层枚举 $j$（到 $M$），内层以 $j$ 为步长跳跃（约 $N/j$ 次）。总循环次数约为：
$$N \times \left(1 + \frac{1}{2} + \dots + \frac{1}{M}\right) \approx N \ln M$$
这就是经典的**调和级数复杂度**，总时间复杂度仅为 $O(N \log M)$，可以完美通过极限数据！

---

### 踩坑与细节预警
1. **减法取模出负数**：由于前缀和数组已经取过模，`Pref[R] - Pref[L-1]` 在相减时可能产生负数，必须写成 `(Pref[R] - Pref[L-1] + MOD) % MOD`。
2. **数据溢出隐患**：$A_i \times i$ 等极端情况极易溢出 32 位 `int` 的上限，强烈建议全员开启 `long long` 防御，并步步取模。
3. **数组越界防范**：处理块的左边界 $L = k \cdot j$ 时，内层循环的 $k$ 应直接从 $1$ 开始，避免 $k=0$ 时导致 `Pref[L-1]` 变成 `Pref[-1]` 从而引发段错误。($k = 0$完全可以省去,因为$k = 0$的时候,结果为0,但是按理来说第一次写的话应该是会写上,这样更加符合逻辑思维的完整性)

### 核心代码

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MOD = 998244353;
const int MAXN = 5e5 + 10;
const int MAXM = 5e5 + 10;
vector <long long> sum_a (MAXN, 0);

int main ()
{
    ios::sync_with_stdio (false);
    cin.tie (0);

    int n, m;
    cin >> n >> m;

    long long lep = 0; // 计算Aii的累加值
    long long rip = 0; // 计算Bj的累加值
    vector <long long> a (n + 1);
    vector <long long> b (m + 1);

    for (int i = 1; i <= n; i ++) {
        cin >> a[i];
        sum_a[i] = (sum_a[i - 1] + a[i]) % MOD;
        lep = (lep + a[i] * i) % MOD;
    }
    for (int i = 1; i <= m; i ++) {
        cin >> b[i];
        rip = (rip + b[i]) % MOD;
    }

    long long all_L = (lep * rip) % MOD;

    // 第二部分
    long long all_R = 0;
    for (int j = 1; j <= m; j ++) {
        long long Bjj = (b[j] * j) % MOD;

        long long Ai_i_j = 0;
        for (int val = 1; val * j <= n; val ++) {// 保证区间左端点小于n就行
            int L = val * j;
            int R = min (n, (val + 1) * j - 1);
            // 在L到R范围内的a[i]的值都是val

            long long range_a = (sum_a[R] - sum_a[L - 1]) % MOD;
            Ai_i_j = (Ai_i_j + range_a * val) % MOD;
        }
        all_R = (all_R + Bjj * Ai_i_j) % MOD;
    }

    long long ans = (all_L - all_R + MOD) % MOD;
    cout << ans << '\n';
    return 0;
}
```
