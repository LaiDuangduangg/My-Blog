## 问题描述

有一个 $H \times W$ 的网格。起初，每个格子里都写着 `A`。

位于第 $i$ 行（从上往下数）、第 $j$ 列（从左往右数）的格子记作 $(i, j)$。

接下来要按顺序执行 $Q$ 次操作。

第 $i$ 次操作会把左上角为 $(1, 1)$、右下角为 $(R_i, C_i)$ 的矩形区域内所有格子的内容，全部改写成大写英文字母 $X_i$。

请输出执行完所有操作后的整个网格。

## 约束条件

- $1 \le H, W$
- $H \times W \le 10^6$
- $1 \le Q \le 2 \times 10^5$
- $1 \le R_i \le H$
- $1 \le C_i \le W$
- $X_i$ 是大写英文字母。
- 所有输入数字均为整数。

## 输入格式

输入从标准输入读入，格式如下：

```text
H W Q
R_1 C_1 X_1
R_2 C_2 X_2
...
R_Q C_Q X_Q
```

## 输出格式

输出 $H$ 行。

第 $i$ 行应包含一个长度为 $W$ 的字符串，其中第 $j$ 个字符表示执行完所有操作后，格子 $(i, j)$ 中的大写英文字母。

```text

```

### 输出


## 题解

根据题意，我们很容易可以知道：

一个点最后会变成什么字符，一定是由**最后一次影响到它的操作**决定的。

每一次操作会把左上角为 $(1,1)$，右下角为 $(R_i,C_i)$ 的矩形全部改成字符 $X_i$。

也就是说，对于某个格子 $(x,y)$，如果一次操作 $(R_i,C_i,X_i)$ 满足：

$$
R_i \ge x,\quad C_i \ge y
$$

那么这次操作就会影响到这个格子。

所以，格子 $(x,y)$ 的最终答案，就是所有能影响到它的操作中，编号最大的那一次操作对应的字符。

---

## 核心思路

我们可以定义一个二维数组 `last`：

```cpp
last[i][j]
```

表示：

> 以 $(i,j)$ 为左上角，往右下方扩展的区域中，出现过的最大操作编号。

换句话说，`last[i][j]` 最后要表示：

> 能够影响到格子 $(i,j)$ 的所有操作中，编号最大的那一次操作。

如果 `last[i][j] = 0`，说明没有任何操作影响过这个格子，那么它仍然是初始字符 `A`。

---

## 操作记录方式

对于第 $k$ 次操作：

```text
R_k C_k X_k
```

我们先不直接修改整个矩形。

因为直接修改矩形会超时。

我们只需要把这次操作记录在它的右下角位置：

```cpp
last[R_k][C_k] = k;
ch[k] = X_k;
```

其中：

- `last[R_k][C_k] = k` 表示第 $k$ 次操作的右下角在 $(R_k,C_k)$；
- `ch[k] = X_k` 表示第 $k$ 次操作对应的字符是 $X_k$。

## 为什么可以这样转移？

如果某次操作的右下角是 $(R,C)$，那么它会影响所有满足：

$$
i \le R,\quad j \le C
$$

的格子。

也就是说，这个操作会影响它左上方的所有格子。

所以对于某个点 $(i,j)$ 来说，它需要知道：

- 自己这个位置有没有操作；
- 它下方区域有没有操作；
- 它右方区域有没有操作。

因此可以从右下角开始，往左上角进行状态转移。

---

## 状态转移

那我们如何考虑状态转移呢？因为每个点要知道从它开始的整个右下角矩形范围的最大值,那推到最后,就变成最右下角那个点也需要知道。
因此我们直接从最右下角那个点开始转移矩形区域最大值即可,那最右下角那个点就只能转移它自己。

对于每个格子 $(i,j)$：

```cpp
last[i][j] = max({
    last[i][j],
    last[i + 1][j],
    last[i][j + 1]
});
```

含义是：

- `last[i][j]`：当前位置本身记录的操作；
- `last[i + 1][j]`：下方能影响到当前位置的最大操作编号；
- `last[i][j + 1]`：右方能影响到当前位置的最大操作编号。

取三者最大值，就能得到影响当前格子的最后一次操作。

扫描顺序必须是：

```text
从右下到左上
```

也就是：

```cpp
for (int i = h; i >= 1; i--) {
    for (int j = w; j >= 1; j--) {
        ...
    }
}
```

因为当前位置需要依赖：

```text
下方的状态 last[i + 1][j]
右方的状态 last[i][j + 1]
```

所以要先把右边和下面算出来。

---

## 输出答案

最后遍历每一个格子：

```cpp
int op = last[i][j];
```

如果：

```cpp
op == 0
```

说明没有任何操作影响这个格子，那么输出：

```cpp
'A'
```

否则输出第 `op` 次操作对应的字符：

```cpp
ch[op]
```


## 复杂度分析

读入操作需要 $O(Q)$。

二维后缀最大值转移需要遍历整个网格，因此是 $O(HW)$。

最后输出答案也是 $O(HW)$。

所以总时间复杂度为：

$$
O(HW + Q)
$$

空间复杂度为：

$$
O(HW + Q)
$$

---

## 参考代码

```cpp
#include <bits/stdc++.h>
using namespace std;

// 二维压一维
long long id(int i, int j, int bases) {
    return i * bases + j;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(0);

    int h, w, q;
    cin >> h >> w >> q;

    int bases = w + 10;

    vector<int> last((h + 10) * (w + 10), 0);
    vector<char> ch(q + 1);

    for (int i = 1; i <= q; i++) {
        int r, c;
        char x;
        cin >> r >> c >> x;

        last[id(r, c, bases)] = i;
        ch[i] = x;
    }

    // 从右下角开始做二维后缀最大值
    for (int i = h; i >= 1; i--) {
        for (int j = w; j >= 1; j--) {
            last[id(i, j, bases)] = max({
                last[id(i, j, bases)],
                last[id(i + 1, j, bases)],
                last[id(i, j + 1, bases)]
            });
        }
    }

    for (int i = 1; i <= h; i++) {
        for (int j = 1; j <= w; j++) {
            int op = last[id(i, j, bases)];

            if (op == 0) cout << 'A';
            else cout << ch[op];
        }
        cout << '\n';
    }

    return 0;
}
```
