## Description

若一个数（首位不为零）从左向右读与从右向左读都一样，我们就将其称之为回文数。

例如：给定一个十进制数 $56$，将 $56$ 加 $65$（即把 $56$ 从右向左读），得到 $121$ 是一个回文数。

又如：对于十进制数 $87$：

STEP1: $87 + 78 = 165$

STEP2: $165 + 561 = 726$

STEP3: $726 + 627 = 1353$

STEP4: $1353 + 3531 = 4884$

在这里的一步是指进行了一次 $N$ 进制的加法，上例最少用了 $4$ 步得到回文数 $4884$。

写一个程序，给定一个 $N$（$2 \le N \le 10$ 或 $N = 16$）进制数 $M$（$100$ 位之内），求最少经过几步可以得到回文数。如果在 $30$ 步以内（包含 $30$ 步）不可能得到回文数，则输出 `Impossible!`。

## Input

两行，分别是 $N$，$M$。

## Output

如果能在 $30$ 步以内得到回文数，输出格式形如 `STEP=ans`，其中 ans 为最少得到回文数的步数。

否则输出 `Impossible!`。

## Sample 1

**Input**

```text
10
87
```

**Output**

```text
STEP = 4
```

## 题解思路
### 1. 预处理：把字符变成真正的“数字”

不要在整个运算过程中留着 `'A'` 到 `'F'` 这种字符。在读入字符串后，第一时间把它映射成 `std::vector<int>`。

*   `'0'` - `'9'` 映射为 $0$ - $9$
*   `'A'` - `'F'` 映射为 $10$ - $15$

这样一来，无论是 $2$ 进制还是 $16$ 进制，在你的数组里都只是纯粹的数值，后续的加法逻辑将完全统一。

### 2. N 进制的高精度加法

回想一下我们学过的高精度加法模板，唯一的区别仅仅是**进位条件**。

平时我们写十进制高精度是 `满 10 进 1`，现在题目给了进制 $N$，那就是 `满 N 进 1`：

*   当前位的值：`(a[i] + b[i] + carry) % N`
*   进位的值：`(a[i] + b[i] + carry) / N`

### 3. 倒序存储（高精度起手式）

为了方便处理最高位的进位，我们通常把数字倒过来存放在 `vector` 中。即 `a[0]` 存放个位，`a[1]` 存放十位。这样遇到进位时，直接往 `vector` 的末尾 `push_back()` 就可以了。

## 易错思路提醒！！！
* 我一开始还想着,两个数组用`高精度加完`之后，然后再`转成字母`的形式，然后`再去判断它是不是回文`。
* 判断完回文之后，`再转成数字数组`，去进行`下一轮的加和比较`。但这个思路是完全没必要的，如果往这个方向想的话，很容易就会因为操作过多然后把自己弄得很乱，搞不好还会崩盘。


## 代码部分:
```cpp
#include <bits/stdc++.h>
using namespace std;

string b = "0123456789ABCDEFG";
map <char, int> find_nums;

// 把初始的进制数直接转成vector + 数字
vector <int> to_nums (vector <char> &a) {
    vector <int> ans;

    for (int i = 0; i < a.size(); i ++) {
        ans.push_back (find_nums[a[i]]);
    }
    return ans;
}

vector <int> add_toge (vector <int> &a, vector <int> &b, int base) {
    vector <int> ans;

    int t = 0; // 进位数
    for (int i = 0; i < min (a.size(), b.size ()); i ++) {
        t += a[i];
        t += b[i];
        ans.push_back (t % base);
        t /= base;
    }
    if (t) ans.push_back (t);
    reverse (ans.begin (), ans.end ());
    return ans;
}

bool check (vector <int> &a) {
    int n = a.size ();
    int i = 0;
    int j = n - 1;

    while (i <= j) {
        if (a[i] != a[j]) {
            return false;
        }
        i ++;
        j --;
    }
    return true;
}

int main ()
{
    int base;
    string s;
    cin >> base >> s;

    //初始化find_nums
    for (int i = 0; i < b.length(); i ++) {
        find_nums[b[i]] = i;
    }

    vector <char> ori;
    for (int i = 0; i < s.length (); i ++) {
        ori.push_back (s[i]);
    }
    vector <int> ack = to_nums (ori);

    if (check (ack) == true) {
        cout << "STEP" << "=" << 1 << '\n';
        return 0;
    } 

    for (int t = 1; t <= 30; t ++) {
        vector <int> fan = ack;
        reverse (fan.begin (), fan.end ());

        vector <int> add_onese = add_toge(ack, fan, base);
        bool is_huiwen = check (add_onese);

        if (is_huiwen == true) {
            cout << "STEP" << "=" << t << '\n';
            return 0;
        }

        // 下一轮让这个ack等于这个加和之后的回文
        ack = add_onese;
    }
    cout << "Impossible!" << '\n';
    return 0;
}
```
