### 问题描述

有 $N$ 张牌和 $N$ 堆牌。

牌和堆的编号都是 $1, 2, \dots, N$。一开始，第 $i$ 堆只包含牌 $i$。

依次执行以下操作，对每个 $i = 1, 2, \dots, Q$：

- 把牌 $C_i$ 以及叠在它上面的所有牌，保持顺序，移动到牌 $P_i$ 的上面。保证在操作前，牌 $C_i$ 和 $P_i$ 在不同的堆里，且牌 $P_i$ 是某堆的最顶牌。

求所有操作完成后，每堆牌的数量。

### 约束条件

- $1 \le N, Q \le 3 \times 10^5$
- $1 \le C_i, P_i \le N$
- 操作按顺序执行时，每次操作前，牌 $C_i$ 和 $P_i$ 在不同的堆里。
- 操作按顺序执行时，每次操作前，牌 $P_i$ 是某堆的最顶牌。
- 所有输入值均为整数。


### 输入格式

从标准输入读取，格式如下：
```text
N Q
C₁ P₁
C₂ P₂
⋮
C_Q P_Q
```


### 输出格式

设 $A_i$ 是最终第 $i$ 堆牌的数量，按顺序输出 $A_1, A_2, \dots, A_N$，用空格分开。











## 🎯 核心知识点

本题的考点为 **双向链表（利用数组进行静态模拟）**。

在算法竞赛中，遇到带有「连续块移动」、「合并」、「插入」等操作，且数据范围达到 $10^5$ 级别时，动态维护每个元素的**绝对位置**是必死之局。这道题完美的展示了如何通过维护元素的**相对位置（上下连接关系）**，将 $O(N)$ 的搬运操作降维打击到 $O(1)$。

## 💡 通解思路与套路（The Playbook）

在 C++ 的竞技编程中，这类题目的通解往往遵循以下三步思考法则：

### 1. 识别“离线查询”特征
仔细审题可以发现，题目是在**“所有操作完成后”**才要求输出每堆牌的张数。中间并不关心某张牌在哪。
**👉 套路结论：** 过程不需要保持完美状态，只需要快速记录每次操作带来的“结构性变化”，最后花 $O(N)$ 的时间整体跑一遍图或链表统计答案。

### 2. 寻找“不变的相对关系”
当把牌 $C_i$ 以及上面的所有牌搬到 $P_i$ 上时，这一大摞牌内部的上下关系变了吗？**完全没有。**
真正发生改变的连接点只有两处：
- $C_i$ 原本下面的牌（如果有的话），失去了 $C_i$。
- $P_i$ 的上面，多出了 $C_i$。
**👉 套路结论：** 抛弃结构体或真实的指针，直接开两个数组 `up[N]` 和 `down[N]` 记录每张牌上面是谁、下面是谁。这就是最纯粹的静态双向链表。每次操作只需修改常数个指针：$O(1)$ 搞定。

### 3. 挖掘题目的“隐藏不变量”（破局关键）
最后怎么知道哪条链属于哪个牌堆？
注意题目的初始条件：“最初，堆 $i$ 仅包含卡片 $i$”，且牌只能往其他牌**上面**放。
这意味着：**如果牌堆 $i$ 里面还有牌，那么压在最底下的一定是卡片 $i$ 本身。**
一旦卡片 $i$ 被搬走了（它的 `down` 指针不再为空），说明牌堆 $i$ 的底座被抽空了，这个牌堆以后将**永远是空的**（因为你无法往空堆里放牌，只能往现存的牌上放）。
**👉 套路结论：** 牌堆 $i$ 的大小，就等于顺着卡片 $i$ 的 `up` 指针一路向上遍历的节点数（前提是卡片 $i$ 的 `down` 指针为空，即它还在最底下）。

## 💻 C++ 核心代码实现

不需要复杂的指针，在竞赛中用一维数组维护双向链表是效率最高、最不易出错的做法。

```cpp
#include <iostream>
#include <vector>

using namespace std;

int main() {
    // 优化输入输出流，CF/AtCoder 必备习惯
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    
    int N, Q;
    if (!(cin >> N >> Q)) return 0;
    
    // up[i] 记录卡片 i 上面的卡片编号
    // down[i] 记录卡片 i 下面的卡片编号
    // 0 表示没有卡片（悬空）
    vector<int> up(N + 1, 0);
    vector<int> down(N + 1, 0);
    
    for (int i = 0; i < Q; ++i) {
        int c, p;
        cin >> c >> p;
        
        // 1. 打断 C_i 与其下方卡片的连接
        if (down[c] != 0) {
            up[down[c]] = 0; 
        }
        
        // 2. 将 C_i 放到 P_i 上面，建立新的连接
        down[c] = p;
        up[p] = c;
    }
    
    // 统计每堆的答案
    vector<int> ans(N + 1, 0);
    
    // 遍历每张卡片，寻找作为“底座”的卡片
    for (int i = 1; i <= N; ++i) {
        // 只有当卡片 i 的下方没有卡片时，它才可能是一个牌堆的基础
        // 根据题意，如果卡片 i 还在最下面，那它一定属于牌堆 i
        if (down[i] == 0) { 
            int size = 0;
            int curr = i;
            // 顺藤摸瓜，往上统计整条链的长度
            while (curr != 0) {
                size++;
                curr = up[curr];
            }
            ans[i] = size;
        }
    }
    
    // 输出结果
    for (int i = 1; i <= N; ++i) {
        cout << ans[i] << (i == N ? "" : " ");
    }
    cout << "\n";
    
    return 0;
}
```
