## 题目描述

某冰箱给每瓶饮料分配了一个 6 位数字字符串作为它的 ID。

这个冰箱里存了 `N` 瓶饮料。第 `i` 瓶饮料的 ID 是 `S_i`，容量是 `V_i`。`S_1, ..., S_N` 各不相同。

现在有 `Q` 个查询。每个查询给出两个 6 位数字字符串 `x`、`y`，要求：

- 在冰箱中找出所有满足以下条件的饮料 ID `s`，并将它们的容量相加：
  - 对于每个 `k = 1, ..., 6`，都有 `x_k ≤ s_k ≤ y_k`。

其中，`s_k`、`x_k`、`y_k` 分别表示字符串 `s`、`x`、`y` 的第 `k` 位数字。

## 约束条件

- `N` 是整数。
- `1 ≤ N ≤ 3 × 10⁵`
- `S_i` 是由数字 `0-9` 组成的字符串。
- `|S_i| = 6`
- `S_1, ..., S_N` 互不相同。
- `V_i` 是整数。
- `1 ≤ V_i ≤ 10⁹`
- `Q` 是整数。
- `1 ≤ Q ≤ 3 × 10⁵`
- 每个查询中的 `x` 和 `y` 都是由数字 `0-9` 组成的字符串。
- 每个查询中，`|x| = |y| = 6`。

## 输入格式

输入从标准输入读入，格式如下：

```text
N
S_1 V_1
...
S_N V_N
Q
query_1
...
query_Q
```

其中，第 `q` 个查询 `query_q` 的格式如下：

```text
x y
```

## 输出格式

输出 `Q` 行，第 `q` 行是第 `q` 个查询的答案。

## 一、大致题意
一句话概括：

> 每个六位 ID 是一个六维点，每次询问要求统计一个六维矩形中的所有容量之和。

设一个 ID 为：

```text
s = s1s2s3s4s5s6
```

一次询问给出：

```text
x = x1x2x3x4x5x6
y = y1y2y3y4y5y6
```

要求每一位都满足：

```text
x1 <= s1 <= y1
x2 <= s2 <= y2
...
x6 <= s6 <= y6
```

注意，这不是普通的数值区间：

```text
x <= s <= y
```

而是六个位置分别进行比较。

---

## 二、暴力模拟

对于每个询问 `(x,y)`：

1. 枚举所有饮料；
2. 检查当前 ID 的六位是否都满足条件；
3. 如果满足，就把容量加到答案中。

一次查询最多检查 `N` 瓶饮料，因此复杂度是：

```text
O(N)
```

总复杂度是：

```text
O(NQ)
```

题目中：

```text
N,Q <= 3×10^5
```

所以最坏大约需要检查 `9×10^10` 次，显然无法接受。

---

## 三、感觉有点像前缀和

先回忆一维区间和。

如果有数组 `a`，定义前缀和：

```text
P(t) = a[0] + a[1] + ... + a[t]
```

那么区间 `[l,r]` 的和就是：

```text
P(r) - P(l-1)
```

本题的每一位数字也只有 `0~9`，所以可以把每个 ID 看成一个点：

```text
(s1,s2,s3,s4,s5,s6)
```

例如：

```text
314159
```

对应六维坐标：

```text
(3,1,4,1,5,9)
```

一次询问要求的就是这个六维矩形：

```text
[x1,y1] × [x2,y2] × ... × [x6,y6]
```

因此它和二维矩形求和非常相似，只不过从二维变成了六维。

---

## 四、先回顾二维前缀和

二维数组中，定义：

```text
P[i][j]
```

表示左上角 `(0,0)` 到 `(i,j)` 的总和。

经典递推是：

```text
P[i][j]
= A[i][j]
+ P[i-1][j]
+ P[i][j-1]
- P[i-1][j-1]
```

为什么要减去左上角？

因为：

- `P[i-1][j]` 统计了上方区域；
- `P[i][j-1]` 统计了左方区域；
- 左上角 `P[i-1][j-1]` 被重复统计了两次。

所以要减掉一次。

---

## 五、二维前缀和的另一种做法

二维前缀和不一定要直接使用四项公式，也可以分两次处理。

### 第一次：每一行做前缀和

```cpp
for (int i = 0; i < 10; ++i) {
    for (int j = 1; j < 10; ++j) {
        A[i][j] += A[i][j - 1];
    }
}
```
处理之后：

```text
A[i][j] = 原来这一行中列 0~j 的总和
```
这一步我们可以理解为,在一个维度上(也就是j这个维度)-1做求和,对照上面式子看就是`A[i][j - 1]`

### 第二次：每一列做前缀和

```cpp
for (int j = 0; j < 10; ++j) {
    for (int i = 1; i < 10; ++i) {
        A[i][j] += A[i - 1][j];
    }
}
```

处理之后：

```text
A[i][j] = 所有 p<=i 且 q<=j 的 A[p][q] 之和
```
这一步我们可以理解为,在另一个维度上(也就是i这个维度)-1做求和,对照上面式子看就是`A[i - 1][j]`
这两种写法得到的结果完全一样。

## 六、低维到高维的推广

六维和二维的区别只有一个：

```text
二维有两个方向；
六维有六个方向。
```

我们不在一个位置一次性处理六个维度，而是：

```text
先让所有位置沿第1维形成前缀；
再让所有位置沿第2维形成前缀；
接着让所有位置沿着第3维度形成前缀
...
最后处理第6维。
```

也就是：

```text
处理完整个第1维
处理完整个第2维
处理完整个第3维
处理完整个第4维
处理完整个第5维
处理完整个第6维
```

### 为什么不在每个位置直接处理六个维度？

不是数学上做不到，而是会非常麻烦。

二维中，一个位置的前缀和只需要处理：

```text
2^2 = 4
```
个角。

三维中，一个位置需要处理：

```text
2^3 = 8
```
个角。

六维中，一个位置需要处理：

```text
2^6 = 64
```

个角。

也就是说，如果直接写六维当前位置的递推公式，每个位置都要写 64 项，代码很长，也非常容易出错。

而分维度处理时，每次只做最简单的一维转移：

```text
当前位置 += 当前维度前一个位置
```

这正是二维“先处理行，再处理列”的推广。

---

## 七、六维前缀和的朴素写法

如果不考虑数组压缩，沿第一维处理就是：

```cpp
for (int d1 = 1; d1 < 10; ++d1) {
    for (int d2 = 0; d2 < 10; ++d2) {
        for (int d3 = 0; d3 < 10; ++d3) {
            for (int d4 = 0; d4 < 10; ++d4) {
                for (int d5 = 0; d5 < 10; ++d5) {
                    for (int d6 = 0; d6 < 10; ++d6) {
                        A[d1][d2][d3][d4][d5][d6]
                            += A[d1 - 1][d2][d3][d4][d5][d6];
                    }
                }
            }
        }
    }
}
```
这里很明显能看到,只在`d1`这个维度继承, 因此是`d1 - 1`

沿第二维处理就是：

```cpp
for (int d1 = 0; d1 < 10; ++d1) {
    for (int d2 = 1; d2 < 10; ++d2) {
        for (int d3 = 0; d3 < 10; ++d3) {
            for (int d4 = 0; d4 < 10; ++d4) {
                for (int d5 = 0; d5 < 10; ++d5) {
                    for (int d6 = 0; d6 < 10; ++d6) {
                        A[d1][d2][d3][d4][d5][d6]
                            += A[d1][d2 - 1][d3][d4][d5][d6];
                    }
                }
            }
        }
    }
}
```
这里就是在`d2`这个维度继承，因此是`d2 - 1`

后面四个维度完全一样。

## 八、六维压一位(压维处理)

代码中使用：

```cpp
prefix[id]
```

而不是：

```cpp
prefix[d1][d2][d3][d4][d5][d6]
```

这是因为六位数字可以唯一对应一个整数：

```text
(d1,d2,d3,d4,d5,d6)
```

对应：

```text
d1*100000 + d2*10000 + d3*1000
+ d4*100 + d5*10 + d6
```

例如：

```text
(3,1,4,1,5,9)
```

对应：

```text
314159
```

这只是存储方式变化，逻辑仍然是六维。

---

## 九、`step` 数组的含义

```cpp
int step[6] = {100000, 10000, 1000, 100, 10, 1};
```

例如：

```text
314159 - 100000 = 214159
```

表示第一位减一。

```text
314159 - 1000 = 313159
```

表示第三位减一。

```text
314159 - 1 = 314158
```

表示第六位减一。

所以：

```cpp
prefix[id] += prefix[id - step[j]];
```

等价于六维数组中的：

```cpp
A[d1][...][dj][...][d6]
    += A[d1][...][dj-1][...][d6];
```

## 十、容斥原理消除重复部分

假设已经有了：

```text
P(b1,b2,b3,b4,b5,b6)
```

它表示：

```text
所有 s_i <= b_i 的饮料容量总和
```

但题目要求的是：

```text
x_i <= s_i <= y_i
```

先取最大范围：

```text
s1<=y1, s2<=y2, ..., s6<=y6
```

也就是：

```text
P(y1,y2,y3,y4,y5,y6)
```

然后排除不满足下界的情况：

```text
s1 < x1
s2 < x2
...
s6 < x6
```

因为数字是整数：

```text
s_i < x_i
```

等价于：

```text
s_i <= x_i-1
```

所以每一维都有两种边界：

```text
y_i
x_i-1
```

六维一共：

```text
2^6=64
```

个组合。

选择下界的次数：

```text
偶数次：加
奇数次：减
```

这就是容斥原理。

## 十一、`mask` 的作用

```cpp
for (int mask = 0; mask < 64; ++mask)
```

`mask` 的每一位代表一个维度。

约定：

```text
第 j 位为 0：这一维使用 y[j]
第 j 位为 1：这一维使用 x[j]-1
```

二维时可以写成：

```text
mask = 00：P(y1, y2)
mask = 01：P(x1-1, y2)
mask = 10：P(y1, x2-1)
mask = 11：P(x1-1, x2-1)
```

六维只是把这件事扩展成 64 种。

代码：

```cpp
if (mask & (1 << j)) {
    cur_pos = x[j] - '0' - 1;
    cnt_choose++;
}
else {
    cur_pos = y[j] - '0';
}
```

如果当前 `mask` 的第 `j` 位是 `1`，就使用：

```text
x[j]-1
```

否则使用：

```text
y[j]
```

---

## 十二、要记得去判断 `cur_pos < 0`
与二维前缀和相同,如果一个前缀到了0边界，要注意-1的越界情况
如果：

```text
x[j] = 0
```

那么：

```text
x[j]-1 = -1
```

它表示：

```text
第 j 位 <= -1
```

但数字只能是 `0~9`，所以这一项对应的前缀区域为空，贡献为 `0`。

因此代码直接跳过：

```cpp
if (cur_pos < 0) {
    valid = false;
    break;
}
```

## 十三、重新计算 `id`

确定了六个边界以后，需要把它们重新拼成一个六位数字。

例如边界是：

```text
0, 6, 9, 9, 4, 4
```

代码：

```cpp
id = 0;
id = id * 10 + 0; // 0
id = id * 10 + 6; // 6
id = id * 10 + 9; // 69
id = id * 10 + 9; // 699
id = id * 10 + 4; // 6994
id = id * 10 + 4; // 69944
```

逻辑上它对应：

```text
069944
```

所以：

```cpp
prefix[id]
```

就是：

```text
P(0,6,9,9,4,4)
```

## 代码如下：
```cpp
#include <bits/stdc++.h>
using namespace std;

constexpr int MAX_RANGE = 1000000;
constexpr int MASKS = 1 << 6;
long long prefix[MAX_RANGE];
int step[] = {100000, 10000, 1000, 100, 10, 1}; // 取位


bool check (const string &x, const string &y) {
    for (int i = 0; i < 6; i ++) {
        int num_x = x[i] - '0';
        int num_y = y[i] - '0';
        if (num_x > num_y) {
            return false;
        }
    }
    return true;
}

int main ()
{
    ios::sync_with_stdio (false);
    cin.tie (0);

    int n;
    cin >> n;
    for (int i = 0; i < n; i ++) {
        string s; int v;
        cin >> s >> v;

        prefix[stoi(s)] += v;
    }
    for (int j = 0; j < 6; j ++) {
        for (int id = 0; id < MAX_RANGE; id ++) { // 六个维度累加
            int cur_dig = id / step[j] % 10;

            if (cur_dig > 0) {
                prefix[id] += prefix[id - step[j]];
            }
        }
    }
    int q;
    cin >> q;
    for (int i = 0; i < q; i ++) {
        string x, y;
        cin >> x >> y;

        bool no_mistakess = check (x, y);
        if (!no_mistakess) {
            cout << 0 << '\n';
            continue;
        }

        long long ans = 0;
        // 每个维度都可以选择y[i] ||x[i] - 1
        // 令 1 表示x[i] - 1
        for (int mask = 0; mask < MASKS; mask ++) {
            bool valid = true;
            int id = 0;
            int cnt_choose = 0;

            for (int j = 0; j < 6; j ++) {
                int cur_pos;
                if (mask & (1 << j)) { // 
                    cur_pos = x[j] - '0' - 1;
                    cnt_choose ++;
                }
                else {
                    cur_pos = y[j] - '0';
                }

                if (cur_pos < 0) {
                    valid = false;
                    break;
                }
                id = id * 10 + cur_pos;
            }
            if (!valid)  continue;
            if (cnt_choose % 2 == 0) {
                ans += prefix[id];
            }
            else {
                ans -= prefix[id];
            }
        }
        cout << ans << '\n';
    }   
    return 0;
}
```

## 十五、复杂度分析

状态总数为：

```text
10^6
```

预处理六个维度：

```text
O(6×10^6)
```

每次查询枚举：

```text
2^6=64
```

个容斥项，因此总复杂度：

```text
O(6×10^6 + 64Q)
```

空间复杂度：

```text
O(10^6)
```
---

核心套路就是：

> 多维点集 + 小值域建表 + 逐维前缀和 + `2^d` 容斥查询。  
>
