## 题目描述

黑板上写有一个整数 $X$。

接下来有 $Q$ 条查询需要依次处理。第 $i$ 条查询，满足 $1 \le i \le Q$，如下：

> 给出两个整数 $A_i$ 和 $B_i$，将这两个新整数 $A_i$ 和 $B_i$ 写到黑板上。  
> 然后，输出黑板上 $2i + 1$ 个整数的中位数。

## 约束条件

- $1 \le X \le 10^9$
- $1 \le Q \le 2 \times 10^5$
- $1 \le A_i, B_i \le 10^9$
- 所有输入均为整数。

## 输入格式

输入从标准输入按如下格式给出：

```txt
X
Q
A_1 B_1
A_2 B_2
:
A_Q B_Q
```

## 输出格式

输出 $Q$ 行。

第 $i$ 行应包含第 $i$ 条查询的答案。

## 样例 1

### 输入

```txt
5
3
2 3
1 2
8 9
```

### 输出

```txt
3
2
3
```

第一次查询后，黑板上的数字变成了 $5, 2, 3$，中位数是 $3$。

第二次查询后，黑板上的数字变成了 $5, 2, 3, 1, 2$，中位数是 $2$。

第三次查询后，黑板上的数字变成了 $5, 2, 3, 1, 2, 8, 9$，中位数是 $3$。

## 样例 2

### 输入

```txt
1
4
2 3
4 5
6 7
8 9
```

### 输出

```txt
2
3
4
5
```

## 题解思路

由于查询次数很多，如果每次都把所有数字排序，肯定会超时。

我一开始想过用「二分 + 插入」的做法来维护序列，但是插入本身仍然需要移动元素，复杂度不够优秀，最后还是会 TLE。

这道题更适合用 **双堆维护中位数**。

## 双堆维护中位数

我们用两个堆来维护当前黑板上的所有数字：

- `pq1`：大根堆，维护较小的一半数字。
- `pq2`：小根堆，维护较大的一半数字。

让 `pq1` 的大小始终等于 `pq2`，或者比 `pq2` 多一个元素。

这样一来，因为当前总数字个数始终是奇数，所以中位数就是：

```cpp
pq1.top()
```

也就是左边较小一半里面的最大值。

## 图示

假设当前有数字：

```txt
1 2 3 4 5
```

我们可以把它们分成两部分：

```txt
左边较小的一半              右边较大的一半

pq1 大根堆                  pq2 小根堆

   _______                     _______
  |   3   |                   |   4   |
  |-------|                   |-------|
  |   2   |                   |   5   |
  |-------|                   |_______|
  |   1   |
  |_______|

      ↑
   中位数
```

也可以理解成：

```txt
pq1: 3 2 1                  pq2: 4 5
     ↑
   中位数
```

所以此时中位数就是 `pq1.top() = 3`。




## 插入操作

每次查询会加入两个数 `A_i` 和 `B_i`。

对于每一个新加入的数字 `x`，我们先判断它应该放在哪个堆里：

- 如果 `x <= pq1.top()`，说明它属于较小的一半，放入 `pq1`。
- 否则，说明它属于较大的一半，放入 `pq2`。

也就是：

```cpp
if (x <= pq1.top()) {
    pq1.push(x);
} else {
    pq2.push(x);
}
```

## 调整两个堆的大小

插入之后，两个堆的大小可能会失衡，所以需要重新调整。

我们希望始终满足：

```txt
pq1.size() == pq2.size()
```

或者：

```txt
pq1.size() == pq2.size() + 1
```

也就是说，`pq1` 最多只能比 `pq2` 多一个元素。

如果 `pq1` 太大：

```cpp
if (pq1.size() > pq2.size() + 1) {
    pq2.push(pq1.top());
    pq1.pop();
}
```

如果 `pq2` 比 `pq1` 大：

```cpp
if (pq2.size() > pq1.size()) {
    pq1.push(pq2.top());
    pq2.pop();
}
```

调整完成后，中位数仍然是：

```cpp
pq1.top()
```

## 完整流程

初始时，黑板上只有一个数字 `X`，直接放入 `pq1`：

```cpp
pq1.push(X);
```

之后每次读入 `A_i` 和 `B_i`：

1. 把 `A_i` 插入到合适的堆里。
2. 调整两个堆的大小。
3. 把 `B_i` 插入到合适的堆里。
4. 再次调整两个堆的大小。
5. 输出 `pq1.top()`。

## 复杂度分析

每个数字只会被插入堆中一次，调整时也只是堆顶元素在两个堆之间移动。

堆的单次操作复杂度是：

```txt
O(log Q)
```

每次查询加入两个数字，所以总复杂度为：

```txt
O(Q log Q)
```

可以通过本题的数据范围。

## 代码如下:
```cpp
#include <bits/stdc++.h>
using namespace std;

priority_queue<long long> pq1;
priority_queue<long long, vector<long long>, greater<>> pq2;

void add (long long x) {
    if (pq1.empty() || x <= pq1.top()) {
        pq1.push(x);
    } else {
        pq2.push(x);
    }

    if (pq1.size() > pq2.size() + 1) {
        pq2.push(pq1.top());
        pq1.pop();
    }
    if (pq1.size() < pq2.size()) {
        pq1.push(pq2.top());
        pq2.pop();
    }
}

int main() {
    ios::sync_with_stdio (false);
    cin.tie (0);

    long long mid;
    int q;
    cin >> mid >> q;    
    add(mid);

    while (q --) {
        long long A, B;
        cin >> A >> B;

        add(A);
        add(B);

        cout << pq1.top() << '\n';
    }
    return 0;
}
```
